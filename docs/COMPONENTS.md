# COMPONENTS.md — KrishiOx

> Documentation-only artifact cataloguing the UI/logic components that already exist in the
> codebase, where they're defined, and where they're used. No components were added, removed,
> or changed to produce this document.

There is no component framework here (no React/Vue/Web Components) — a "component" in this
document means a reusable chunk of markup + CSS + JS convention that appears identically (or
near-identically) across pages, generated by a shared function in `js/main.js`/`js/icons.js`,
or styled by a shared CSS block in `style.css`.

## 3. Booking wizard components (`booking.html` + `js/booking.js`)

The wizard is the most stateful part of the codebase. All of the following are rendered by
functions inside the `js/booking.js` IIFE, gated on `state.step`:

| Step | Component | Render function | Validates via |
|---|---|---|---|
| Progress bar | `#wizardProgress` — 6 filled/active segments | `renderProgress()` | — |
| 1. Service | Service card grid (tap-to-advance) | `renderServiceGrid()` | `state.serviceId` must be set |
| 2. Village | Text input + `<datalist>` + quick-select chips | `renderVillageStep()` / `syncVillageChips()` | non-empty trimmed `state.village` |
| 3. Quantity/Details | Stepper (+/−) for normal services, or a textarea for "अन्य सेवा" (Other) | `renderQuantityStep()` | qty > 0, or non-empty `otherDetails` for "other" |
| 4. Date | Quick-date chips (आज/कल/2 दिन बाद/अगले सप्ताह) + native `<input type="date">` | `renderDateStep()` / `syncDateChips()` | `state.date` set |
| 5. Remarks | Name (required), phone (optional, 10 digits if present), free-text remarks | `renderRemarksStep()` | name non-empty; phone empty or exactly 10 digits |
| 6. Confirm | Editable summary rows (each with a "बदलें"/change link back to its step) | `renderSummary()` | — (final step, triggers `confirmBooking()`) |
| Success screen | `#successWrap` — replaces the wizard after confirm | `confirmBooking()` | — |

Shared wizard chrome: `.wizard-nav` (Back/Next buttons, `updateNavButtons()` relabels Next to
"WhatsApp पर बुक करें" on the last step), step labels (`updateStepLabels()`), and draft
persistence (`saveDraft()`/`restoreDraftIfAny()`/`clearDraft()`, backed by `KrishiOxStore`).

## 5. Icon library (`js/icons.js`)

Two categories of keys in `KRISHIOX_ICONS`:

- **Service icons** (10): `tractor`, `rotavator`, `cultivator`, `laser`, `harvest`, `loader`,
  `trolley`, `transport`, `irrigation`, `labour`, `other` — one per `KRISHIOX_SERVICES` entry,
  matched by the `icon` field on each service object.
- **UI icons** (~24): navigation (`home`, `grid`, `calendar`, `chat`, `handshake`), actions
  (`check`, `checkCircle`, `plus`, `refresh`, `install`, `back`, `chevronRight`,
  `arrowRight`), and contextual icons (`whatsapp`, `phone`, `bell`, `shield`, `clock`,
  `rupee`, `map`, `target`, `star`, `location`, `future`, `rotate`, `menu`, `info`).

`krishiOxIcon(name)` falls back to the `other` icon (three dots) for any unrecognized key —
this is the only defensive-default in the icon system, and it fails silently rather than
throwing, which is worth knowing if a typo'd icon name ever ships.
